# TP01 · 数据层

> 扩展动作数据模型，并把 [docs/fitness/index.md](../../fitness/index.md) 的 86 个精选动作生成为 Swift 内置数据。这是后面所有页面的基础，**最先做**。

## 目标

- `Exercise` 模型支持：分类、动作类型、难度、男女视频（占位）、肌群（用于解剖图筛选）。
- 86 个动作全部录入，分 7 类。
- 提供按「分类 / 肌群 / 类型 / 关键词」查询的入口，供 TP02/TP04 调用。

## 数据来源与字段映射

每个动作在 index.md 表格里是一行：

| index.md 列 | → Swift 字段 |
|---|---|
| 动作（英文） | `nameEn` |
| 中文名 | `name` |
| 主要肌群 | `primaryMuscles`（顿号分隔 → `[String]`） |
| 类型 | `type`（如「卧推/胸推」「下拉」「引体向上」） |
| 难度 `n/5` | `difficulty: Int`（1–5） |
| Female / Male 链接 | `femaleVideo` / `maleVideo`（占位期存文件名或空串） |
| 所在小节 | `category`（7 类枚举） |

7 类：核心与腹部 `core` / 胸部与推 `chest` / 背部与拉 `back` / 肩部 `shoulders` / 手臂与握力 `arms` / 下肢与臀腿 `lower` / 全身功能 `functional`。

## 模型设计（建议）

```swift
enum MuscleCategory: String, CaseIterable, Identifiable {
    case core, chest, back, shoulders, arms, lower, functional
    var displayName: String { /* 核心 / 胸 / 背 / 肩 / 手臂 / 下肢 / 功能 */ }
    var id: String { rawValue }
}

/// 用于解剖图高亮 + 筛选的肌群 slug（对齐 react-native-body-highlighter 的 slug，便于 TP04 复用 path）
enum MuscleGroup: String, CaseIterable {
    case lats, traps, rhomboids, lowerBack          // 背
    case chestUpper, chestMid, chestLower            // 胸（或合并 chest）
    case absRectus, obliques, transverse             // 核心
    case deltsFront, deltsSide, deltsRear            // 肩
    case biceps, triceps, forearms                   // 臂
    case quads, glutes, hamstrings, calves, adductors// 腿
    var displayName: String { /* 中文名 */ }
}

struct Exercise: Identifiable {
    let id = UUID()
    let name: String
    let nameEn: String
    let category: MuscleCategory
    let primaryMuscles: [String]      // 展示用中文，如 ["背阔肌"]
    let muscleGroups: [MuscleGroup]   // 解剖图筛选用，主练在前
    let type: String
    let difficulty: Int               // 1–5
    let maleVideo: String             // 占位期可空
    let femaleVideo: String
    // 要点/组次：index.md 没有，可后补；先给空数组/空串，详情页做空态降级
    let points: [String]
    let setsReps: String
}
```

> `points`（动作要点）和 `setsReps` 在 index.md 里**没有**——旧 4 部位数据里有手写要点。处理方式二选一（在本任务里定）：
> - A：先留空，详情页对空要点做降级（不显示「动作要点」区）。**推荐，先跑通。**
> - B：从旧 `TrainingPlanData` 里能对上的动作迁移要点，其余留空。

## 生成方式（避免手敲 86 行出错）

写一次性脚本解析 index.md 表格 → 生成 Swift 数组源码，人工核对后落库。

- 输入：`docs/fitness/index.md`（小节标题定 `category`，表格行定字段）。
- 难度从 `n/5` 取 `n`；视频列从 markdown 链接里抽文件名（`female/xxx.mp4`），URL decode。
- `muscleGroups` 由「主要肌群」中文 → slug 的映射表推导（映射表手工定一份，约 20 条肌群即可覆盖）。
- 输出：`TrainingPlanData.exercises: [Exercise]`，写进 `TrainingPlanModels.swift`。
- 脚本可放 `scratchpad`，不入库；只入库生成结果。

## 查询接口（供后续页面用）

```swift
extension TrainingPlanData {
    static func exercises(in category: MuscleCategory) -> [Exercise]
    static func exercises(for muscle: MuscleGroup) -> [Exercise]
    static func search(_ keyword: String) -> [Exercise]   // 匹配 name / nameEn / 肌群
    static func types(in category: MuscleCategory) -> [String]  // 该分类下的动作类型，用于 chips
}
```
