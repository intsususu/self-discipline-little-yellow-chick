## 页面结构（自上而下）

1. **导航头**：返回 + 动作中文名。
2. **视频区（占位）**：16:10 区域。
   - 占位期：深色底 + 居中播放图标。**无手动男女切换**。
   - 演示性别按个人资料自动匹配：`isFemale = profileStore.profile.gender == .female`（`.female` → 女版，其余 → 男版）。由主页透传 `isFemale`，或本页自建 `@StateObject ProfileStore()` 读取。
   - 数据有 `maleVideo`/`femaleVideo` 时按性别取；为空（5 个无视频动作）显示「演示视频待补充」。
   - 预留接入点：`AVPlayer`/`VideoPlayer`，素材就位后替换占位（见 README 视频口径）。
3. **标题块**：中文名 + 英文名 + 右侧难度徽章「难度 n/5」。
4. **标签行**：主练肌群（`主 · 背阔肌`，accent 底）+ 类型 chip。
5. **肌肉发力图**：调用 [TP04 解剖图](TP04-可点击解剖图.md) 的人体视图，**高亮该动作的 `muscleGroups`**（静态展示，不可点）。
   - TP04 未完成前：用占位图 `TrainingIllustrationView`（现有组件）降级。
6. **动作要点**：bullet 列表 + 右上角组次徽章。
   - `points` 为空时（TP01 选方案 A）**隐藏整区**，不留空标题。
7. **底部免责声明**。

## 与解剖图组件的关系

详情页的发力图 = TP04 里 `MuscleMapView` 的**只读模式**（传入 `highlight: [MuscleGroup]`，`interactive: false`）。建议 TP04 先把人体视图抽成可复用组件 `MuscleBodyView(highlighted:onTap:)`，详情页和解剖图页共用。
