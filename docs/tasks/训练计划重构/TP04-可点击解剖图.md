# TP04 · 可点击解剖图

> 对应原型「③ 解剖图选肌群」。本任务工作量最大、可独立交付，建议主流程跑通后再做。
>
> 依赖：[TP01 数据层](TP01-数据层.md)。

## 目标

- 渲染**正面 + 背面**人体（占位期男版即可，男/女可后做），各肌群是独立可着色区块。
- 点某块肌群 → 高亮 + 列出该肌群的所有动作（`TrainingPlanData.exercises(for:)`）。
- 抽出**可复用人体视图** `MuscleBodyView(highlighted:onTap:)`，详情页（只读高亮）和本页（可点）共用。

## 人体形状从哪来

移植 [HichamELBSI/react-native-body-highlighter](https://github.com/HichamELBSI/react-native-body-highlighter)（MIT）的 path 数据：

- 文件：`assets/bodyFront.ts` / `bodyBack.ts`（男）、`bodyFemaleFront.ts` / `bodyFemaleBack.ts`（女）。
- 每个肌群形如 `{ slug, path: { left: ["M…"], right: ["M…"] } }`，slug 对齐 TP01 的 `MuscleGroup`。
- viewBox 约 `0 0 660 1500`（以实际文件为准）。

**渲染两条路线，二选一：**

### 路线 A：PocketSVG（省事，推荐先试）
- SPM 接入 [PocketSVG](https://github.com/pocketsvg/PocketSVG)（MIT）。
- 把每个肌群 path 字符串 → `SVGBezierPath` → `Path`（SwiftUI）或 `CAShapeLayer`。
- 每块肌群一个 `Shape`，按是否高亮填色；点击用 `.contentShape(path).onTapGesture`。

### 路线 B：自写极小 path 解析器（零依赖）
- 只需支持 path 命令子集（M/m L/l C/c Q/q Z 等，body 数据基本是这些）。
- 写 `func parseSVGPath(_ d: String) -> Path`，约 100–150 行。
- 优点：不引依赖，符合 README「不引运行时依赖」倾向；缺点：要自己测命令覆盖。

> 建议：先用路线 A 快速验证形状/坐标正确，确认数据可用后，若想去依赖再换路线 B（path 数据不变，只换解析）。

## 数据准备

- 把 TS 里的 path 数组导出成 App 能读的格式（如 `BodyPaths.swift` 常量，或一份 `body_front.json` 放 bundle）。
- 维护 `MuscleGroup ↔ slug` 映射（TP01 已用同名 slug，尽量直接对齐）。
- 注意 left/right 两侧 path，渲染时都画。

## 交互

- `MuscleBodyView(highlighted: Set<MuscleGroup>, onTap: (MuscleGroup)->Void)`。
- 解剖图页：tap 肌群 → 设为 selected → 下方列出动作（复用 TP02 的 `ExerciseRow`）。
- 正/背面切换或并排展示（原型是并排两个小人）。
- 详情页：`onTap = nil`，仅高亮 `exercise.muscleGroups`。

## 验收

- [ ] 正/背面人体能正确渲染（无路径错位/破面）。
- [ ] 点任一肌群有高亮反馈，并列出对应动作；空肌群给空态文案。
- [ ] `MuscleBodyView` 被详情页只读复用成功。
- [ ] 移植的 path 数据在代码注释/`THIRD-PARTY-NOTICES` 标注 MIT 来源。
- [ ] 若用 PocketSVG：依赖加入 SPM 且真机编译通过。
