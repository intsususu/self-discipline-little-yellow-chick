# TP02 · 动作库主页

> 对应原型「① 动作库主页」。重写 [TrainingPlanView.swift](../../../HealthApp/Features/Tools/TrainingPlan/TrainingPlanView.swift)，从 4 部位图鉴改成动作库浏览页。
>
> 依赖：[TP01 数据层](TP01-数据层.md)。

## 页面结构（自上而下）

1. **导航头**：标题「训练计划」。**不放男女切换**——演示性别按个人资料自动匹配（见下「演示性别」）。
2. **搜索框**：占位「搜索动作 / English」，输入即过滤（调 `TrainingPlanData.search`）。有输入时下方直接出搜索结果列表，隐藏 tabs/chips。
3. **解剖图入口卡**：正/背面小人体缩略 + 文案「点身体部位筛选动作」，点击进入 [TP04 解剖图页](TP04-可点击解剖图.md)。
   > TP04 未完成前，此卡先做成静态占位（不可点或 toast），不阻塞主页交付。
4. **部位 tabs**（横向胶囊，单选）：核心 / 胸 / 背 / 肩 / 手臂 / 下肢 / 功能，默认「背」。复用现有 `pill` 样式。
5. **类型 chips**：当前分类下的动作类型（`TrainingPlanData.types(in:)`），含「全部」，单选过滤。
6. **动作卡列表**：见下。
7. **底部免责声明**（沿用现有 `disclaimer`）。

## 动作卡（列表项）

紧凑横向卡片（比旧的大卡更密）：

- 左侧 56×56 缩略区：占位期放播放图标 `figure...` / `play`（视频就位后换首帧）。
- 右侧：中文名（15–16）+ 英文名（11，muted）+ 一行标签（主练肌群 chip + 类型 chip + 难度圆点 `●●○`）。
- 整卡可点 → push [TP03 详情页](TP03-动作详情页.md)。

> 可把列表项抽成 `ExerciseRow`，与旧 `ExerciseCard`（大卡）区分；旧 `ExerciseCard` 若详情页复用则保留，否则删。

## 演示性别（不手动切换）

- 来源：`@StateObject private var profileStore = ProfileStore()`，读 `profileStore.profile.gender`。
- 规则：`.female` → 女版；`.male` / `.other` / 未填（默认即 `.male`）→ 男版。
- 主页列表项缩略图就位后据此取男/女首帧；占位期不影响。
- 派生：`var isFemale: Bool { profileStore.profile.gender == .female }`，透传给详情页（TP03）。

## 状态与交互

- `selectedCategory`、`selectedType`、`searchText` 用 `@State`。性别不存本页状态，直接读 `profileStore`。
- 切分类时重置 `selectedType` 为「全部」，列表滚回顶部（沿用现有 `ScrollViewReader` + topAnchor 方案）。
- 保留左右滑动切分类的手势（现有 `pageSwipeGesture`）。
- 搜索为空 → 显示 tabs+chips+分类列表；非空 → 显示全局搜索结果。

## 难度圆点

`difficulty` 1–5 → `●` 实心 ×n + `○` 空心 ×(5-n)，或简化成 3 档色（易/中/难）。和原型一致用圆点即可，颜色用暖色（`.exerciseOrange` 系）。
