# 健康数据分析 App · 产品需求文档（PRD）

> 版本：v1.0 ｜ 日期：2026-06-19 ｜ 平台：iOS（SwiftUI 原生）
> 来源：根据 `健康数据分析App-线框图.html`（低保真）与 `健康App-高保真原型-离线版.html`（可交互高保真原型）整理。
> 用途：供 Claude / Codex 等工具**直接施工**。所有数值、颜色、文案均来自原型，作为实现的数据契约。

---

## 1. 概述

### 1.1 产品定位
一款 **iOS 个人健康数据分析 App**：导入 Apple 健康（HealthKit）数据后，围绕**体重（核心）/ 睡眠 / 运动**三个维度做趋势与关联分析；并允许用户记录**特殊事件**（生病、运动损伤、饮酒、旅行/出差），把这些事件**叠加到图表**上，用来解释数据的异常波动。

### 1.2 目标用户
- 正在减脂 / 自律管理身材的人群（原型用户画像：178cm·34 岁·男·减脂中）。
- 已经在用 Apple 健康记录体重/睡眠/运动，但缺少「趋势 + 事件归因」分析的人。

### 1.3 核心价值
1. **以体重为中心**的长期趋势可视化（周/月/年/全部）。
2. **事件归因**：把「感冒那周体重回升 0.6kg」「饮酒当晚深睡 ↓」这类因果，用图表标注直观呈现——这是与系统自带「健康」App 的主要差异点。
3. **隐私优先**：全部分析在本机完成，数据不出设备。

### 1.4 平台与技术栈
| 项 | 选择 | 说明 |
|---|---|---|
| 平台 | iOS 16.0+ | Swift Charts 需要 iOS 16 |
| UI | SwiftUI | 原生，Tab 导航 |
| 图表 | Swift Charts | 折线/柱状/睡眠阶段 |
| 数据 | HealthKit | 阶段二接入；阶段一用 Mock |
| 架构 | MVVM + Repository | 见 §9 |
| 存储 | 本机（事件用 SwiftData / Core Data） | 不上云 |

## 2. 范围（Scope）

### 2.1 分阶段
- **阶段一（Mock UI 完整版）**：用本文 §6 提供的 mock 数据集，搭出全部界面、图表、交互、事件记录与叠加分析。`HealthDataRepository` 由 `MockHealthRepository` 实现。**此阶段即为可演示的完整 App。**
- **阶段二（接入 HealthKit）**：新增 `HealthKitRepository` 实现同一协议，替换数据源；新增 A3 导入/授权流程。UI 层不改。

### 2.2 MVP 包含
- 5 个 Tab：总览 / 体重 / 睡眠 / 运动 / 我的。
- 事件与周期记录模块（记录 + 时间轴 + 图表叠加 + 关联洞察）。
- Apple 健康导入/授权界面（阶段二联调）。

### 2.3 MVP 不包含（见 §11 Backlog）
- 推送提醒（仅在「我的」展示设置项占位）。
- 数据导出落地（仅入口）。
- Apple Watch / Widget。
- 多用户、账号体系、云同步。

## 3. 信息架构与导航

底部 Tab 导航（经典 iOS），全局右上角「＋ 记事件」入口。

```
App
├── Tab 1 总览 (Home)           A1
│     └─ ＋记事件 → 事件记录弹窗 (E2)
├── Tab 2 体重 (Weight)         A2  →  体重·事件视图 (E3)
├── Tab 3 睡眠 (Sleep)          A4
├── Tab 4 运动 (Exercise)       A5
├── Tab 5 我的 (Profile)        A6
│     └─ ＋记录特殊事件 → 事件时间轴 (E1) / 记录弹窗 (E2)
└── 全局组件
      ├─ 记录事件弹窗 (E2 modal) —— 任意 Tab 右上＋ 均可唤起
      └─ Toast「已记录：…」
```

- **总览**给一眼概况；4 个数据 Tab 各自钻深。
- 每个数据 Tab 顶部有**时间范围切换**（见 §8）。
- 事件保存后：关闭弹窗 → 顶部 Toast → 相关图表自动出现事件标注。

## 5. 功能需求（按屏）

> 每屏标注对应线框图编号（A*/E*）与高保真原型。所有文案/数值为默认 mock 值。

### 5.1 Tab1 · 总览仪表盘（A1）
**用途**：一眼看到今日概况与本周亮点。
**UI 组成**：
- 顶部：日期「6月18日 星期四」+ 问候语「早上好，今天」；右上「＋ 记事件」按钮（pill）。
- **体重 Hero 卡**（`weightCardBg`）：当前体重 `77.1 kg`；右上「本周 ↓0.9 / 累计 -13.9」；底部周趋势 sparkline；副信息「距目标 73kg · 还差 4.1kg」「查看趋势 ›」。
- **3 指标圆环卡**：睡眠 `7.3h`、运动 `68m`、卡路里 `434 千卡`。
- **本周亮点 / 关联洞察卡**（虚线框）：如「体重连续 4 周下降，运动负荷与睡眠时长同步走高 👍」；高保真版含事件洞察「感冒发烧那周运动暂停，体重回升 0.6kg…」。
- **近期事件卡**：最近事件列表 + 「＋ 记录」。
**交互**：点 Hero 卡 → 体重 Tab；点「＋ 记事件」→ E2 弹窗。

### 5.2 Tab2 · 体重（A2 / E3）
**用途**：体重长期趋势 + 事件叠加分析（核心页）。
**UI 组成**：
- 标题「体重」+ 右上「＋」。
- **时间范围切换**：周 / 月 / 年 / 全部。
- **主折线图**：体重(kg)，含**目标线 73**（虚线 + 标注），含 `◆ 事件` 标记开关。
- **事件影响条**：如「5月31日 · 感冒发烧 — 病后一周运动暂停，体重回升 0.6kg。点查看 ›」。
- **统计三元组**：当前 `77.1` / 累计 `-13.9` / 历史最低 `71.9`（高保真另有历史最高 `91.2`）。
- **最近记录列表**：6月15日 77.1kg / 6月08日 78.0kg / 6月01日 78.9kg …
- **E3 事件视图**：开关「在图上显示事件」→ 单日事件=点标记，时间段事件=色带；图例（生病/损伤(段)/饮酒/旅行(段)）；底部可点事件行 + 关联洞察「拉伤那周运动暂停，体重回升 0.6kg；出差期间作息乱、下降也停滞。」
**交互**：切换时间范围重绘；点事件标记/事件行 → 事件详情/影响。

### 5.3 Tab3 · 睡眠（A4）
**用途**：睡眠时长、效率、阶段分析。
**UI 组成**：
- 时间范围切换：周/月/年。
- **睡眠 Hero**：平均睡眠 `7.3h`，`↑0.2h`，效率 `95%`（高保真：夜醒 8.3 次）。
- **睡眠阶段（日均）**：深睡 `0.65h(41分)` / 核心 `4.67h` / REM `1.58h(100分)` / 清醒 `0.36h`；阶段条形/环形。深睡「偏少·建议↑」，REM「正常区间」。
- **本周每晚时长柱状图**：一~日。
- **事件影响**：如「6月7日 · 饮酒（聚餐）— 当晚深睡 ↓、清醒增多，睡眠效率降到 88%。」

### 5.4 Tab4 · 运动（A5）
**用途**：运动消耗、时长、心率与类型。
**UI 组成**：
- 时间范围切换；高保真支持指标切换（千卡 / 心率）+ 近 6 个月。
- **运动 Hero**：日均消耗 `434 千卡`，本周 5 次 / 日均 68 分。
- **统计三元组**：日均时长 `68 分` / 有氧心率 `121 bpm` / 周消耗 `3.0 千`（高保真：累计 `39.5 千`千卡）。
- **每日时长柱状图**（分钟）。
- **运动类型**：跑步 3 · 力量 1 · 骑行 1（高保真：有氧占 66%，累计燃脂约 5.1kg）。
- **事件影响**：「5月20–27日 · 腰肌肉拉伤 — 期间运动暂停，周消耗降到平时 1/3。」

### 5.5 Tab5 · 我的（A6）
**用途**：用户信息、目标、设置。
**UI 组成**：
- 头像 + 「李 · 减脂中」+「178cm · 34 岁 · 男 · 已连接 Apple 健康」。
- **目标体重卡**：`73.0 kg`，「还差 4.1kg」，可「编辑」。
- 设置组 1：数据来源「Apple 健康 ✓」/ 单位「公斤(kg) ›」/ 每日提醒「8:00 ›」。
- 设置组 2：＋记录特殊事件「生病/损伤/饮酒/旅行 ›」/ 导出数据 › / 隐私与安全「本机存储 ›」/ 关于「v1.0 ›」。
**交互**：编辑目标体重（影响全局目标线/距目标计算）；记录特殊事件 → E1/E2。

### 5.6 导入 Apple 健康（A3）— 阶段二
**用途**：首次授权并同步历史数据。
**UI 组成**（卡片式 3 步）：
1. 授权读取权限：体重 · 睡眠 · 运动 · 心率。
2. 同步历史数据：首次导入约 2019 年至今。
3. 生成分析报告：趋势 · 关联 · 建议。
- 主按钮「连接 Apple 健康」；次按钮「稍后手动导入」。
- 文案：「授权后我们将读取体重、睡眠与运动记录，全部分析在本机完成。」

### 5.7 事件时间轴（E1）
**用途**：集中查看所有特殊事件。
**UI 组成**：
- 标题「事件」+ 右上「＋」。
- **类型筛选 chips**：全部/生病/损伤/饮酒/旅行。
- 月份分组「2026 年 6 月」。
- **时间轴 rail**：每条事件含 类型色点 + 标题 + 单日/时间段标签 + 日期 + 备注。例：
  - 出差·上海（时间段）6月10–14日·5天「作息紊乱，运动暂停」
  - 饮酒·聚餐（单日）6月07日
  - 感冒发烧（单日）5月31日·已就医
  - 腰肌肉拉伤（时间段）5月20–27日

### 5.8 添加 / 编辑事件（E2，全局弹窗）
**用途**：记录某一天或一段时间的特殊事件。
**UI 组成**：
- 顶部：取消 / 标题「新建事件」/ 保存。
- **类型选择**：生病 / 损伤 / 饮酒 / 旅行（/ 其他）。
- **持续切换**：单日 / 时间段（选时间段显示开始 + 结束两个日期）。
- **日期**：默认 `2026年6月18日`，日历选择。
- **标题**（可选，默认取类型名）。
- **备注**：占位「添加描述，例如"感冒发烧，停训"…」。
- 提示「记录后会在体重/睡眠/运动图表上标注，帮你解释数据波动。」
**交互**：保存 → 新事件插入列表顶部 → 关闭 → Toast「已记录：{标题}」（约 2.2s 自动消失）→ 图表叠加更新。

## 6. 数据模型

> 阶段一直接内置以下 mock 数据集（来自原型，**请原样使用**）。

### 6.1 Swift 结构体（建议）
```swift
// 体重样本（单点）
struct WeightSample: Identifiable {
    let id = UUID()
    let date: Date      // 测量日
    let kg: Double
}

// 睡眠样本（单晚）
struct SleepSample: Identifiable {
    let id = UUID()
    let date: Date
    let totalMinutes: Int        // 当晚总时长（分）
    var deepMinutes: Int? = nil  // 深睡
    var coreMinutes: Int? = nil  // 核心
    var remMinutes: Int? = nil   // REM
    var awakeMinutes: Int? = nil // 清醒
    var efficiency: Double? = nil // 效率 %
}

// 运动样本（按月聚合或按次）
struct ExerciseSample: Identifiable {
    let id = UUID()
    let label: String   // 月份/日期标签
    let kcal: Double     // 消耗千卡
    let avgHR: Double?   // 平均心率
    let minutes: Int?    // 时长
}

// 事件类型
enum EventType: String, CaseIterable, Codable {
    case illness, injury, drink, travel, other
    var label: String { ["illness":"生病","injury":"损伤","drink":"饮酒","travel":"旅行","other":"其他"][rawValue]! }
    // color/bg 见 §4.2
}

// 特殊事件（单日 or 时间段）
struct HealthEvent: Identifiable, Codable {
    let id: String
    var type: EventType
    var title: String
    var startDate: Date        // 单日时 == 该日
    var endDate: Date?         // nil = 单日；非 nil = 时间段
    var note: String
    var isPeriod: Bool { endDate != nil }
}
```

## 7. HealthKit 集成（阶段二）

### 7.1 需读取的类型
| 维度 | HealthKit 类型 |
|---|---|
| 体重 | `HKQuantityType(.bodyMass)` |
| 睡眠 | `HKCategoryType(.sleepAnalysis)`（含 deep/core/REM/awake 分段，iOS 16+） |
| 活动能量 | `HKQuantityType(.activeEnergyBurned)` |
| 运动时长 | `HKQuantityType(.appleExerciseTime)` 或 `HKWorkout` |
| 心率 | `HKQuantityType(.heartRate)` |

### 7.2 授权流程（对应 A3）
1. `HKHealthStore.requestAuthorization(toShare: [], read: <上述类型>)`。
2. 首次同步历史（约 2019 至今）：分维度用 `HKSampleQuery` / `HKStatisticsCollectionQuery` 拉取，按周/月聚合缓存。
3. 生成分析（趋势/关联/建议）。
- 仅读不写；未授权 → 显示空状态并引导重新授权。

### 7.3 聚合
- 体重：周（按 ISO 周取均值或最新）、月、年；用 `HKStatisticsCollectionQuery` 的 `anchorDate + intervalComponents`。
- 睡眠：按「睡眠夜」归集分段，算总时长、效率、各阶段。
- 运动：按月聚合 kcal 与平均心率。

### 7.4 隐私
- 全部本机存储与计算，无网络上传（文案需明确）。
- 事件数据用 SwiftData/Core Data 本机持久化。
- `Info.plist` 添加 `NSHealthShareUsageDescription`。

## 8. 图表与分析逻辑

### 8.1 图表类型
- **体重**：折线（Swift Charts `LineMark` + `AreaMark` 渐变填充）；目标线 `RuleMark`(虚线)；事件叠加。
- **睡眠**：每晚柱状（`BarMark`）；睡眠阶段用堆叠条或环形。
- **运动**：每日/每月柱状；可切 kcal/心率指标。

### 8.2 时间范围切换
| 维度 | 范围 |
|---|---|
| 体重 | 周 / 月 / 年 / 全部 |
| 睡眠 | 周 / 月 / 年（默认近 14 晚/30 天） |
| 运动 | 近 6 个月（指标可切 千卡 / 心率） |

### 8.3 事件叠加规则
- **单日事件**：在折线对应日期画**点标记**（`PointMark`/菱形 `◆`），颜色取事件类型色。
- **时间段事件**：画**色带**（`RectangleMark`/背景区间），半透明类型色 bg。
- 图例：生病 / 损伤(段) / 饮酒 / 旅行(段)。
- 提供「在图上显示事件」开关（E3）。

### 8.4 关联洞察文案规则
基于事件与指标变化生成可读句子（mock 阶段可硬编码，HealthKit 阶段按规则计算）：
- 体重：「{事件}那周运动暂停，体重回升 {Δ}kg；恢复训练后 {n} 周内回落。」
- 睡眠：「{事件}当晚深睡平均减少 {x}%。」
- 运动：「{事件}停训一周，当周消耗降到平时约 1/3。」
计算方式：取事件窗口内 vs 事件前同长度窗口的指标差值。

## 9. 架构建议

### 9.1 模式
SwiftUI + MVVM；数据通过 **Repository 协议**注入，便于 Mock↔HealthKit 切换。

```swift
protocol HealthDataRepository {
    func weightSeries(range: TimeRange) async -> [WeightSample]
    func sleepSeries(range: TimeRange) async -> [SleepSample]
    func exerciseSeries(range: TimeRange) async -> [ExerciseSample]
    func events() async -> [HealthEvent]
    func saveEvent(_ e: HealthEvent) async
}
// 阶段一：MockHealthRepository（内置 §6 数据）
// 阶段二：HealthKitRepository（读 HealthKit）+ EventStore（本机持久化）
```

### 9.2 建议目录结构
```
HealthApp/
├── App/                 // @main, Tab 容器
├── Models/              // WeightSample, SleepSample, HealthEvent, EventType ...
├── Repository/          // HealthDataRepository, Mock*, HealthKit*
├── Features/
│   ├── Home/            // A1 总览
│   ├── Weight/          // A2 / E3
│   ├── Sleep/           // A4
│   ├── Exercise/        // A5
│   ├── Profile/         // A6
│   ├── Events/          // E1 时间轴, E2 记录弹窗
│   └── Import/          // A3 授权（阶段二）
├── Charts/              // 复用图表组件 + 事件叠加
├── DesignSystem/        // Color tokens, Card, RingMetric, Pill...
└── Resources/           // mock json
```

### 9.3 全局状态
- `goalWeight`（默认 73.0，可编辑）。
- `events`（全局，影响所有图表叠加）。
- Toast 服务（保存事件后展示）。
