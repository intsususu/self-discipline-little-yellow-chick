# 综合分析报告设计

> 目标：在「我的 · 数据与偏好」中新增「综合分析」入口，选择时间段（最长 3 个月）后，
> 把这段时间的**减脂、运动、睡眠**数据拉到一起，用「先总结 → 两角度点评 → 暖心寄语收尾」的方式生成一份复盘报告。
> 风格定位：不是冷冰冰的打分，而是一只「自律小黄鸭」教练帮你做复盘。
> 状态：**设计 + 交互原型**，尚未落地代码。交互原型见同目录 [`prototype.html`](./prototype.html)。

---

## 1. 入口与导航

- 位置：`ProfileView` 的「数据与偏好」分组，**「体测数据导入」上方**新增一行 `综合分析`（icon 建议 `chart.line.uptrend.xyaxis` 或小鸭图标）。
- 点击 → `时间段选择页` → 确认 → `报告页`。两页都走「我的」既有的 `NavigationStack` 压栈（系统返回 + 左缘右滑），与 `ImportView` 的二级管理页一致。

## 2. 时间段选择页

| 元素 | 说明 |
|---|---|
| 预设快捷选项 | 近一周 / 近一个月 / 近三个月（默认近一个月） |
| 自定义区间 | 与事件录入共用 `DateRangePickerSection`：开始/结束日期行 + 年/月/日三列轮盘；点选日期后自动收起 |
| **上限校验** | 区间 > 3 个月（约 92 天）时，提示「超过 3 个月上限」并禁用「生成报告」按钮 |
| 数据完整度提示 | 「已选 N 天 · 区间内约有数据 M 天」，避免「选了 90 天但只有 12 天有记录」造成误读 |
| CTA | 底部 `safeAreaInset` 固定「生成报告」主按钮 |
| 生成转场 | 点击后全屏展示 1.5 秒「小鸭教练分析中」+ 程序小鸭头像 + loading 指示器，结束后进入报告页 |

**对比口径**：报告默认与**上一个等长周期**对比（选 7 天 → 对比前 7 天；选 30 天 → 对比前 30 天），用于「做得好 / 要注意」里的「↑↓ 变化」。这是报告的关键设计——所有正/负反馈都来自**跨周期对比**，而非绝对值。

## 3. 报告结构（四段）

报告**不打分**，由四段顺序构成：

### ① 概览总结（事实层，不带评价）
- 使用程序 `AppIcon` 作为小鸭头像，顶部展示「小鸭教练说 / 给你的周期小结」
- **体重块**：周期内体重变化、距离体重目标
- **运动块**：运动总消耗、运动总次数、运动总时间；消耗最大单日（日期、单日总消耗、各运动类型及消耗）；最常运动类型（次数、累计运动时间）
- **睡眠块**：日均睡眠质量评分、最高分、最低分、日均睡眠时长、日均就寝时间、日均起床时间
- 睡眠质量评分复用睡眠页统一算法：时长 50 分 + 规律性 30 分 + 中断 20 分，范围 50–100 分

### ② 做得好的地方（正反馈，绿色 ✓）
对身体有益、或较上一周期有进步的数据。

### ③ 需要注意的地方（风险点，橙色 ⚠）
退步、风险、事件对运动的负面影响。

### ④ 小鸭想对你说（暖心寄语）
按数据情绪选类的一句话，暖黄卡、衬线字体；不展示情绪标签和「换一句」入口。

## 4. 数据来源映射

| 维度 | 数据源 | 用到的字段 |
|---|---|---|
| 减脂 | `WeightSample` + 目标体重 | `date`, `kg` |
| 运动频率 | `WorkoutSession` | `start`, `type`, `minutes`, `kcal` |
| 运动表现 | `WorkoutSession` | `avgHR`, `minutes`, `kcal` |
| 睡眠 | `SleepSample` | `totalMinutes`, `deep/core/rem/awakeMinutes`, `efficiency`, `bedtime`, `wakeTime` |
| 事件归因 | `HealthEvent` | `type`, `startDate`, `endDate`, `isExerciseRelated` |
| 画像 | `UserProfile` | `age`（算最大心率 `220−age`）、`gender` |

> 需在 `HealthDataRepository` 增加**按区间查询**接口（`weights(in:)` / `workouts(in:)` / `sleeps(in:)` / `events(in:)`）。

## 5. 「做得好 / 要注意」生成规则

两个角度的条目**不写死**，由规则引擎扫描候选信号，**触发阈值才纳入**，按显著性排序，**每个角度最多 4 条**（太多会稀释重点）。

### 5.1 候选信号库

| 信号 | 进入「做得好」 | 进入「要注意」 |
|---|---|---|
| 体重 | 净下降、达成阶段目标 | 净上升、连续 2+ 周未降（平台期/反弹） |
| 运动次数 | 较上周期 ↑ | 较上周期明显 ↓ |
| 运动频率（近 2 周） | 维持/上升 | 跌破基线 |
| 连续打卡 | 刷新纪录 | 最长中断变长 |
| 运动强度/效率 | kcal/min ↑、高强度占比 ↑ | 强度整体偏低 |
| 心率适应性 | 同类运动 avgHR ↓ | 同类运动 avgHR ↑ |
| 睡眠规律性 | 入睡波动收窄 | 入睡波动 > 阈值（±45min） |
| 睡眠时长/达标率 | ↑ | 跌破目标、达标率低 |
| 睡眠结构 | 深睡占比 ↑ | 深睡占比 ↓、效率 ↓ |
| 饮酒次数 | 较上周期 ↓ | 较上周期 ↑ |
| 事件影响 | — | 伤病/出行致中断、饮酒致睡眠下降 |

### 5.2 排序与截断
- 每条信号算一个**显著性分**（变化幅度归一化 × 权重）。
- 各角度按显著性降序取前 4。
- 若「做得好」为空（极端情况），降级展示「保持得不错」类兜底文案，避免空卡。

### 5.3 事件归因（重点）
利用 `HealthEvent.isExerciseRelated`：
- **伤病 / 出行**区间内运动次数明显低于周期均值 → 归因为「该事件致运动中断 N 天」，并关联同期体重变化。
- **饮酒**当晚 `SleepSample.efficiency` 低于个人均值 → 归因为「饮酒当晚睡眠效率平均下降 X%」。
- 这样把异常**归到具体事件**，避免用户误以为是自己懈怠。

## 6. 暖心寄语系统（结合数据动态选）

### 6.1 情绪判定
先由本周期数据算一个**整体情绪** `sentiment ∈ {positive, negative, neutral}`：

| 情绪 | 触发条件（示意，可调权重） |
|---|---|
| `positive` 进步期 | 体重净下降 **且** 正反馈条数 ≥ 风险条数 |
| `negative` 反弹期 | 体重上升 **或** 风险条数显著多于正反馈（如多 2 条以上） |
| `neutral` 平稳期 | 体重基本持平（\|Δ\| < 0.5kg）且无明显反弹 |

### 6.2 文案选取
- 按 `sentiment` 选对应**文案分类**，生成报告时取一句；报告页不暴露情绪标签或文案切换操作。
- 三类语气：
  - **positive → 肯定 + 鼓励**
  - **negative → 打气 + 安慰**（不指责，强调「翻开报告本身就是没放弃」）
  - **neutral → 陪伴 + 推一把**
- 文案库每类 6–8 条（原型已各备 6 条），最终建议各扩到 15–20 条降低重复感。

### 6.3 文案库（节选，详见 `prototype.html` 的 `SOUP` 常量）
- positive：「你不是在和体重较劲，而是在和昨天那个想偷懒的自己赛跑。」
- negative：「反弹不可怕，怕的是因为一次后退，就把走过的整条路都否定掉。」
- neutral：「稳住，是一种被低估的能力。接下来，可以试着再推自己一把。」

## 7. 视觉规格（沿用现有设计系统）

| 元素 | 规格 |
|---|---|
| 卡片 | `CardView`：圆角 16、`hairline` 描边、`shadow .04` |
| 小鸭头像 | 直接复用程序 `AppIcon`；概览使用 42pt 圆角头像，寄语标题使用 24pt 圆角头像 |
| 周期小结 | 上方为小鸭头像与「小鸭教练说」；下方按体重绿、运动橙、睡眠紫分为三个纵向数据块 |
| 维度配色 | 体重 `weightGreen #16a34a`、运动 `exerciseOrange`、睡眠 `sleepIndigo` |
| 维度卡背景 | 体重 `weightCardBg`、运动 `exerciseCardBg`、睡眠 `sleepCardBg` |
| 做得好 | 标题/✓ 用 `successGreen` |
| 要注意 | 标题/⚠ 用 `exerciseOrange`；事件来源图标用事件色（出行棕、饮酒紫、伤病红） |
| 寄语卡 | 暖黄底 `#fffaeb` + 描边 `#fce9b8`，衬线字体；仅展示标题与寄语正文 |
| 字重 | 仅 regular / medium（对齐现有页面，不用 700） |
