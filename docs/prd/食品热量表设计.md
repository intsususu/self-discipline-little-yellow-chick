# 食品热量表 PRD（交互设计）

> 目标：在「我的 → 数据与偏好 → 小工具」下提供一个本地、离线的食品热量查询工具，帮助用户快速查到常见食物/饮料/快餐/水果零食的热量参考值。
> 状态：**交互设计已定稿（来自 Claude Design 原型，含本轮调整）**，待落地为 SwiftUI 页面。当前「小工具」分组中「食品热量表」为占位入口（`placeholderToast`），本文档定义其完整交互与内置数据。
> 入口：[ProfileView.swift](../HealthApp/Features/Profile/ProfileView.swift) `toolsSettings` → `食品热量表` 行。

## 1. 范围与定位

- **纯查询工具**：本地内置数据，无需联网。热量为常见做法的**参考值**，非营养学精确值（页面底部明确标注「热量为常见做法参考值，实际以食材与烹饪方式为准」）。
- **本期不做「记一笔到今日饮食」**：详情页**不含**记录/入库按钮。后续若上线饮食记录功能，再单独评估是否在详情页加「记一笔」入口。
- 全中文界面，风格与 App 现有卡片/分组一致。

## 2. 页面结构（单页 + 底部详情弹层）

整页为一个**可滚动列表页**，从上到下：

1. **导航头**：左侧返回箭头（`<`，返回「我的」），居中标题「食品热量表」。
2. **搜索框**：占位文案「搜索食物名称，如 米饭、苹果、奶茶」；左侧放大镜图标；输入后右侧出现清除（×）按钮。
3. **一级分类 Tab（横向）**：`主食 / 快餐 / 水果零食 / 饮料`，单选，选中态为蓝色实心胶囊。默认选中**主食**（无「全部」项）。
4. **二级标签 Tab（条件显示，横向）**：仅当一级分类下存在子类/品牌时出现，左侧带一个标签图标。单选，选中态为深色实心胶囊。
   - `主食`：无二级 Tab（直接展示该分类全部条目）
   - `快餐`：肯德基 / 麦当劳 / 必胜客
   - `水果零食`：水果 / 干果 / 零食
   - `饮料`：常见饮料 / 星巴克 / 瑞幸 / Manner / 库迪
   - 切换一级分类时，二级 Tab 默认选中**第一项**。
5. **结果条**：左侧「共 N 种食物」计数，右侧**排序切换**（带上下箭头图标）。
6. **食物列表**：每行一个食物卡片（见 §3）。
7. **底部免责声明**：「热量为常见做法参考值，实际以食材与烹饪方式为准」。

点击任一食物行 → 从底部滑出**详情弹层**（见 §4）。

> 本轮调整：原一级分类的「水果」「干果」上收，合并为新的一级分类「**水果零食**」，二者降为其内部二级标签，并新增「零食」二级标签。

## 3. 食物列表行

| 元素 | 说明 |
|---|---|
| 图标 | emoji（米饭🍚、可乐🥤等），圆角浅色底 |
| 名称 | 食物名（如「可口可乐」「吮指原味鸡」） |
| 副标题 | `分类/品牌 · 计量单位`，如「常见饮料 · 每罐」「肯德基 · 每块」「主食 · 每碗」 |
| 热量值 | 大号彩色数字 + 「千卡」，颜色随热量等级变化 |
| 等级徽标 | `低 / 中 / 高` 小胶囊，绿 / 黄 / 红三色 |

**热量等级配色**（按**当前展示份量**的热量划分，用于数字颜色 + 徽标）：
- 低（绿）：< 100 千卡
- 中（黄/橙）：100–299 千卡
- 高（红/橙）：≥ 300 千卡

> 阈值落地时可按数据分布微调；建议用 App 既有的绿/黄/红语义色。

## 4. 详情弹层（Bottom Sheet）

从底部滑出，顶部带拖拽手柄，可下滑/点遮罩关闭。内容：

1. **头部**：emoji + 食物名；下方两个标签：分类（如「常见饮料」）+ 热量等级（如「中热量」）。
2. **主热量块**：超大数字 + 「千卡」，右侧标注默认计量单位（如「每罐」）。
3. **常见份量**：列表，每行一个份量规格及其对应热量，例如可口可乐：`1 罐 / 330ml → 140 千卡`、`大瓶 / 500ml → 215 千卡`。
4. **底部说明**：「热量为常见做法参考值，实际以食材与烹饪方式为准」。

> 本期详情弹层**不含**「+ 记一笔到今日饮食」按钮。

## 5. 搜索 / 排序 / 筛选规则

- **搜索**：在**当前一级分类（含二级 Tab）范围内**按食物名匹配过滤；切换分类时搜索词**保留**并在新分类内重新过滤。无结果时展示空状态：放大镜图标 + 「没有找到「关键词」」+ 「换个关键词，或选择上方分类浏览」。
- **排序**：点击循环切换三档 —
  1. `默认排序`（内置顺序）
  2. `热量高 → 低`
  3. `热量低 → 高`
  排序状态在切换分类/二级 Tab 时**保留**。
- **筛选叠加关系**：`一级分类` ∩ `二级 Tab` ∩ `搜索词`，再按当前排序输出。

## 6. 数据模型（建议）

内置静态数据（随包 JSON 或 Swift 字面量），结构示意：

```
FoodCategory   { id, name, subtabs: [FoodSubtab]? }   // subtabs 为空则无二级 Tab
FoodSubtab     { id, name }                            // 如 肯德基 / 水果 / 星巴克
FoodItem       {
  id, name, emoji,
  category, subtab?,                                   // 归属
  unitLabel,           // 默认计量单位文案，如「每碗」「每罐」「每100克」
  calorie,             // 默认份量热量（千卡，整数）
  level,               // 低/中/高（可由 calorie 推导）
  portions: [ { label, spec, calorie } ]              // 常见份量（≥1 条）
}
```

- 新建 `Features/Tools/FoodCalorie/`：`FoodCalorieView.swift`（主页）、`FoodDetailSheet.swift`（详情弹层）、`FoodCalorieStore.swift`（过滤/排序逻辑）、`FoodCalorieData.swift` 或随包 `food_calorie.json`（§7 数据）。

- 复用现有设计令牌：胶囊 Tab、`CardView`、等级配色对齐 App 既有的绿/黄/红语义色。
