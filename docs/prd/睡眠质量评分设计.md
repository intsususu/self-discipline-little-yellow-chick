# 睡眠质量评分设计（对齐 iOS 26 / watchOS 26 Sleep Score）

> 目标：把现有睡眠数据量化为 0–100 的「睡眠质量分」，尽量贴近 Apple iOS 26 / watchOS 26 的 Sleep Score 模型。
> 状态：**设计 + 验算**，尚未落地代码。

## 1. Apple 官方模型

Apple 的 Sleep Score 是 0–100 分，由三个分项加权构成，满分各不同：

| 分项 | 满分 | 衡量对象 |
|---|---|---|
| **睡眠时长** Duration | **50** | 实际**睡着**时长 vs 睡眠目标（不是卧床时长） |
| **就寝一致性** Bedtime Consistency | **30** | 每晚上床时间是否稳定（与自身历史比较） |
| **夜间中断** Interruptions | **20** | 夜里醒来的次数 / 清醒时间 |

**等级带**（watchOS 26.2 / iOS 26.2 起的新划分）：

| 分数 | 等级 |
|---|---|
| 0–40 | Very Low |
| 41–60 | Low |
| 61–80 | OK |
| 81–95 | High |
| 96–100 | Very High |

**官方校准样例**：`40/50（时长）+ 30/30（就寝）+ 14/20（中断）= 84 → High`。下文用它校准曲线。

## 2. 数据可用性盘点

`SleepSample` 现有字段：`totalMinutes`（睡着时长）、`deep/core/rem/awakeMinutes`、`efficiency`、`date`。

| 分项 | 现有数据 | 能否计算 |
|---|---|---|
| 时长 50 | `totalMinutes` ✅ | **可直接算** |
| 中断 20 | `awakeMinutes` ✅ | **可算**（清醒分钟做代理） |
| 一致性 30 | ❌ 每晚上床时刻未存储 | **缺口**，见下 |

**缺口说明**：
- `SleepView` 里的入睡时间 `23:42`、夜醒次数 `8` 是写死的占位值（`HealthApp/Features/Sleep/SleepView.swift:216`）。
- HealthKit 聚合时每段的 `startDate/endDate` 被丢弃，只保留分钟总数（`HealthApp/Repository/HealthKitRepository.swift:573`）。
- 即：**就寝一致性所需的原始时刻 HealthKit 里有，只是没被保留**。补这一项只需在聚合时多留「当晚最早入睡时刻」，无需新权限。

## 3. 子公式（已用 Apple 样例校准）

记号：睡眠目标 `G = 480 分钟`（8h，可配置）；当晚睡着 `T` 分钟、清醒 `A` 分钟；近 7–14 晚上床时刻标准差 `SD` 分钟。

### ① 时长（0–50）

令 `r = T / G`：

```
时长分 = clamp(84·r − 34, 0, 50)
```

- 校准：7h 睡 / 8h 目标 → `r=0.875` → `84×0.875−34 = 39.5 ≈ 40`，命中官方 40/50。
- `r ≥ 1.0` 封顶 50；`r ≤ 0.405` 归零。
- 可选：对严重赖床（`r > 1.2`）施加轻微衰减，初版可不做。

### ② 中断（0–20）

```
中断分 = clamp(24 − 0.4·A, 0, 20)
```

- 校准：要得 14/20 需 `A=25` 分钟清醒，合理。
- `A ≤ 10` 满分；`A ≥ 60` 归零。

### ③ 就寝一致性（0–30）

取近 7–14 晚上床时刻的标准差 `SD`（分钟）：

```
一致性分 = clamp(30·(1 − (SD−15)/75), 0, 30)
```

- `SD ≤ 15min` 满分；`SD ≥ 90min` 归零。
- **当前缺原始数据**：补上 `bedtime` 字段前，暂以代表值或「时长波动」做弱代理。

### 总分

```
睡眠质量分 = 时长分 + 中断分 + 一致性分   // 0–100
```

## 4. 验算（用 mock 真实数据）

数据取自 `HealthApp/Repository/MockHealthRepository.swift:173`，`G=480`，一致性暂用代表值 `SD≈20min → 28 分`。

| 日期 | T(睡着) | A(清醒) | 时长分 | 中断分 | 一致性* | **总分** | 等级 |
|---|---|---|---|---|---|---|---|
| 06-06 好觉 | 515 (r=1.07) | 15 | 50（封顶） | 18 | 28 | **96** | Very High |
| 06-07 饮酒夜 | 464 (r=0.97) | 30 | 47 | 12 | 28 | **87** | High |
| 06-17 短觉 | 335 (r=0.70) | 18 | 25 | 17 | 28 | **70** | OK |

\* 一致性为代表值，补上床时刻后逐晚计算。

**逐项手算（06-07 饮酒夜）**：

- 时长：`84×0.967 − 34 = 47.2 → 47`
- 中断：`24 − 0.4×30 = 12`
- 一致性：`28`
- 合计：`47 + 12 + 28 = 87 → High`

结果分布合理：最佳夜 Very High、饮酒夜 High、短觉夜 OK，与直觉和官方等级带一致，且与官方 84 分样例同处 High 带。

## 5. 落地建议

- **时长(50) + 中断(20) = 70 分**两项现可精确计算，公式已校准。
- **一致性(30)** 是唯一缺口：
  1. 在 `HealthKitRepository.aggregateSleep` 聚合时保留「当晚最早入睡时刻」。
  2. 给 `SleepSample` 增加 `bedtime` 字段。
  3. 用近 7–14 晚 `bedtime` 标准差代入公式 ③。
- 实现顺序建议：
  - **阶段一**：独立 `SleepScore` 纯函数计算器 + 单元测试（含上表用例），先不接 UI、一致性用占位。
  - **阶段二**：补 `SleepSample.bedtime`，让一致性真算。
  - **阶段三**：接入睡眠页 UI，展示分数与等级带。
